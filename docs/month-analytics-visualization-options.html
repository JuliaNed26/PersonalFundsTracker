Visualization options for the new current month analytics page.

This standalone page uses sample data close to your feature spec: current month expenses are compared against the whole previous month, and savings are shown as current-month net saved grouped by currency. Each section below shows one visualization style so you can decide what feels clearest for your app.

Sample assumptions
  • Expenses are shown in one comparison currency: `UAH`.
  • Savings are grouped by their own goal currency.
  • `Net saved` means deposits minus withdrawals this month.
1. Summary + ranked list 2. Grouped bars 3. Dumbbell chart 4. Waterfall chart 5. Donut + delta list 6. Savings cards + progress
Best MVP choice

1. Summary cards + ranked comparison list

This option is the fastest to scan. It tells the user whether spending is up, by how much, and which categories are responsible without forcing them to decode a chart first.

Category comparison

Sorted by biggest month-over-month movement so the most important changes are immediately visible.

Why it works
Very readable on mobile, cheap to build, and friendly for future notifications that deep-link into the page.
Weakness
Less visually dramatic than a chart-heavy screen, especially if you want a more “analytics dashboard” feel.
Strong for comparison

2. Grouped horizontal bar chart

This view makes current versus previous month obvious with almost no explanation. It works especially well when the user wants to spot category spikes quickly.

Expenses by type

Each category shows the previous month bar first, then the current month bar on the same scale.

Elegant comparison

3. Dumbbell chart

The dumbbell chart is compact and expressive. It is excellent when the whole story is “how far did each category move?” and you want less visual bulk than full bars.

Distance between previous and current month

Grey dot is the previous month. Orange dot is the current month. The line between them makes change direction easy to read.

Best for explanation

4. Waterfall chart

This style answers a different question: not just what changed, but which categories built the jump from the previous month total to the current month total.

What caused the increase

Positive deltas push the running total upward. Negative deltas pull it back down.

Best for composition

5. Donut chart + delta list

This option helps when you care about where the current month money is going, not only how much changed. It is strongest as a secondary view rather than the main month-over-month comparison view.

Current month spending mix

Current month across all expense types

Composition with month-over-month delta

Useful if you want to see both share of spend and which categories changed most.

Best for savings

6. Savings currency cards + per-goal progress rows

This is the clearest savings format for your app because savings can exist in different currencies. It avoids misleading totals while still showing momentum on each goal.

Saving goals this month

Each row keeps the goal’s own currency visible and uses progress bars for the monthly target.

Negative rows are useful because they immediately show when a user withdrew from savings during the month.
